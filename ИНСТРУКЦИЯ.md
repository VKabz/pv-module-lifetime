# Инструкция по запуску

Программа рассчитывает срок службы солнечного фотоэлектрического модуля
по климатическим данным региона и строит графики. Ниже — подробная
пошаговая инструкция. Опыт работы с Python не требуется.

## 1. Что нужно для запуска

- Компьютер с **Windows** или **macOS**.
- Установленный **Python 3.10 или новее** (как установить — см. раздел 2).
- **Интернет** при первом запуске (программа скачивает климатические
  данные и библиотеки). Дальше работает без интернета.

---

## 2. Установка Python

> Если Python уже установлен — пропустите этот раздел.

### Windows

1. Откройте сайт https://www.python.org/downloads/
2. Нажмите жёлтую кнопку **«Download Python 3.x.x»**.
3. Запустите скачанный файл.
4. **ВАЖНО:** на первом экране установщика поставьте галочку
   **«Add python.exe to PATH»** (внизу окна), затем нажмите
   **«Install Now»**.
5. Дождитесь окончания установки и нажмите «Close».

### macOS

1. Откройте сайт https://www.python.org/downloads/
2. Нажмите кнопку **«Download Python 3.x.x»**.
3. Откройте скачанный файл `.pkg` и пройдите установку
   (везде «Продолжить» → «Установить»).

Проверить установку можно так: откройте «Терминал» (macOS) или
«Командную строку» (Windows) и введите `python --version` — должна
появиться строка вида `Python 3.12.x`.

## 3. Запуск в один клик

1. Распакуйте папку с программой в удобное место (например, на «Рабочий стол»).
2. Откройте папку.
3. Запустите файл для вашей системы:

   | Система | Файл для запуска |
   |---------|------------------|
   | **Windows** | дважды кликните по `Запустить_расчёт.bat` |
   | **macOS** | дважды кликните по `Запустить_расчёт.command` |

4. Откроется чёрное окно (консоль) — это нормально. В нём будет видно,
   как идёт расчёт.
5. **Первый запуск** длится 1–2 минуты: программа сама устанавливает
   нужные библиотеки и скачивает климатические данные. Последующие
   запуски — несколько секунд.
6. По окончании автоматически откроется папка **`output`** с готовыми
   графиками и таблицами.

> **macOS, первый запуск.** Если система пишет, что файл «от неустановленного
> разработчика» и не запускает его — кликните по файлу
> `Запустить_расчёт.command` **правой кнопкой мыши** → выберите
> **«Открыть»** → в появившемся окне ещё раз **«Открыть»**. Это нужно
> сделать только один раз.

## 4. Что появится в результате

После запуска в папке **`output`** будут такие файлы:

| Файл | Что это |
|------|---------|
| `fig_3_1_degradation.png` | График накопленной деградации по климатическим зонам |
| `fig_3_2_weibull.png` | Кривые выживания модулей (распределение Вейбулла) |
| `fig_3_3_scenarios.png` | Влияние качества модуля (тест DH1000) на срок службы |
| `fig_temp_Москва.png` | Среднемесячная температура воздуха и ячейки |
| `table_3_3_regions.csv` | Сводная таблица по регионам (открывается в Excel) |
| `table_3_5_scenarios.csv` | Таблица сценариев качества модулей (Excel) |

Графики (`.png`) — готовы для вставки в текст работы (разрешение 300 dpi).
Таблицы (`.csv`) открываются двойным кликом в Excel.

В консоли также печатается сводная таблица с результатами для всех
регионов — её можно скопировать.

## 5. Запуск вручную

Если запуск в один клик не сработал, выполните команды вручную.

### Windows (Командная строка)

Откройте папку с программой, в адресной строке проводника введите `cmd`
и нажмите Enter — откроется консоль уже в нужной папке. Затем введите
по очереди:

```bat
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\python main.py
```

### macOS / Linux (Терминал)

Откройте Терминал, перейдите в папку с программой (команда `cd` и путь
к папке), затем:

```bash
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
./.venv/bin/python main.py
```

## 6. Возможные проблемы и решения

**«python не является внутренней или внешней командой» (Windows)**
Python не добавлен в PATH. Переустановите Python и обязательно поставьте
галочку **«Add python.exe to PATH»** (см. раздел 2).

**«command not found: python» (macOS)**
Используйте `python3` вместо `python`.

**Ошибка при скачивании данных / нет соединения**
При первом запуске нужен интернет — программа обращается к серверу
NASA POWER. Проверьте подключение и запустите ещё раз. После первого
успешного запуска данные сохраняются в папку `data` и интернет больше
не нужен.

**Долго «висит» на установке библиотек**
Это нормально при первом запуске (скачивается ~50 МБ библиотек).
Подождите 1–2 минуты.

**Хочу запустить заново «с нуля»**
Удалите папку `.venv` (окружение) и/или папку `data` (кэш данных) и
запустите программу снова.

## 7. Как изменить параметры расчёта

Все настройки собраны в начале файла **`main.py`** (открывается любым
текстовым редактором — «Блокнот», VS Code и т. п.).

**Параметры модуля** (блок `MODULE`):

```python
MODULE = {
    "NOCT": 45,        # номинальная рабочая температура ячейки, °C
    "d_ref": 0.005,    # опорная скорость деградации (0,5 %/год)
    "Ea_ev": 0.55,     # энергия активации, эВ
    "n": 2.5,          # показатель влияния влажности
    "beta": 30,        # угол наклона панели, градусы
    "dP_DH": 0.05,     # деградация в тесте DH1000 (5 %)
}
```

**Список регионов** (блок `REGIONS`) — чтобы добавить регион, допишите
строку с его координатами:

```python
REGIONS = [
    {"name": "Москва", "lat": 55.75, "lon": 37.62, "climate": "Умеренный"},
    # добавьте свой регион по образцу:
    {"name": "Казань", "lat": 55.79, "lon": 49.12, "climate": "Умеренный"},
]
```

**Год климатических данных** (переменная `YEAR`):

```python
YEAR = "2023"   # измените на нужный год
```

После изменения сохраните файл и запустите программу снова.

## Проверка корректности расчётов

В комплект входит скрипт `verify.py` — он независимо пересчитывает
ключевые величины и сверяет их с результатами программы. Запуск:

```bash
# Windows
.venv\Scripts\python verify.py
# macOS
./.venv/bin/python verify.py
```

В конце должно появиться: **«ВСЕ ПРОВЕРКИ ПРОЙДЕНЫ. Расчёты корректны.»**
