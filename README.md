# Прогнозирование срока службы фотоэлектрического модуля

Расчётная часть ВКР: модель прогнозирования срока службы солнечных
фотоэлектрических модулей по климатическим данным региона.

Модель объединяет:
- **линейное накопление деградации** мощности;
- **температурную зависимость Аррениуса** (ускорение деградации при нагреве);
- **влажностную модель Пэка** (ускорение коррозии при высокой влажности);
- **модель температуры ячейки Номера–Пирса** (NOCT);
- **калибровку по стандартному тесту DH1000** (85 °C / 85 % RH / 1000 ч);
- **распределение Вейбулла** для статистического разброса в партии модулей.

Климатические данные берутся из открытого API **NASA POWER** (почасовые,
глобальное покрытие).

## Быстрый запуск (без знания Python)

Нужен только установленный [Python 3](https://www.python.org/downloads/)
(при установке на Windows отметьте галочку **«Add Python to PATH»**).

- **macOS** — дважды кликните по файлу **`Запустить_расчёт.command`**
- **Windows** — дважды кликните по файлу **`Запустить_расчёт.bat`**

Лаунчер сам создаст окружение, установит все библиотеки (при первом
запуске, 1–2 минуты, нужен интернет), выполнит расчёт и откроет папку
`output/` с готовыми графиками и таблицами. Больше ничего делать не нужно.

> На macOS при первом открытии система может предупредить о неизвестном
> разработчике — нажмите правой кнопкой по файлу → «Открыть» → «Открыть».

## Запуск вручную (для разработчиков)

```bash
# 1. Создать окружение и установить зависимости
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 2. Запустить расчёт
python main.py
```

Скрипт скачает климатические данные (при первом запуске), выполнит расчёты
и сохранит результаты в папку `output/`:

| Файл | Содержание |
|------|-----------|
| `fig_3_1_degradation.png` | Кривые накопленной деградации по климатическим зонам |
| `fig_3_2_weibull.png`     | Кривые выживания (распределение Вейбулла) |
| `fig_3_3_scenarios.png`   | Влияние качества модуля (DH1000) на срок службы |
| `fig_temp_Москва.png`     | Среднемесячная температура воздуха и ячейки |
| `table_3_3_regions.csv`   | Сводная таблица по регионам |
| `table_3_5_scenarios.csv` | Таблица сценариев качества модулей |

Скачанные климатические данные кэшируются в `data/`, поэтому повторные
запуски выполняются мгновенно и дают воспроизводимый результат.

## Структура проекта

```
.
├── main.py              # главный скрипт — полный цикл расчёта
├── src/

├── data/                # кэш скачанных климатических данных
├── requirements.txt     # зависимости Python
└── README.md
```

## Настройка под свою работу

Все параметры собраны в начале `main.py`:

- `MODULE` — параметры исследуемого модуля (NOCT, энергия активации `Ea`,
  показатель влажности `n`, угол наклона `beta`, деградация в тесте `dP_DH`);
- `REGIONS` — список регионов (название, широта, долгота, тип климата);
- `YEAR` — год климатических данных.

Чтобы добавить регион, допишите строку в `REGIONS` с его координатами.
